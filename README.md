# 商工中金情報システム 採用サイト リニューアル比較モックアップ

新卒採用ページ（[shochu-comp.co.jp/pg600.html](https://www.shochu-comp.co.jp/pg600.html) および [newgraduate_recruitment](https://www.shochu-comp.co.jp/newgraduate_recruitment/)）を、「仕事意識の高い学生」により響くリニューアル案として検討するための、社内検討用ローカルモックアップです。

## 構成

- `current/` — 現行サイトの構成・トーンを忠実に再現したモックアップ（丸ゴシック・緑基調・安心/受容重視のコピー）
- `new/` — リニューアル案（ダーク×エディトリアル・データと裁量と成長を前面に出す編集的トーン）

どちらもビルド不要の素のHTML/CSS(+最小限のJS)です。各サイトの `index.html` を直接ブラウザで開くか、簡易サーバーで確認できます。

```bash
# current を確認
open current/index.html

# new を確認
open new/index.html

# もしくは簡易サーバーで
python3 -m http.server 8080
# → http://localhost:8080/current/index.html
# → http://localhost:8080/new/index.html
```

## 画像について

実際の商工中金情報システムの写真は一切使用していません。すべての写真枠はCSSグラデーションのプレースホルダーです（詳細は各サイトの `images/README.md` を参照）。

## データ・デザインの正確性について（v2で全面改訂）

`current/` は当初テキストベースの調査のみで作成したため実サイトと見た目が大きく異なっていました。その後、実際に商工中金情報システムの各ページ（`pg600.html` と `newgraduate_recruitment/` 配下の各ページ）をブラウザで開いて配色・フォント・レイアウト・実数値を直接確認し、以下の点を全面的に修正しています。

- **2つの異なるテンプレートを区別**：`current/index.html` は本体サイトの採用ハブページ（pg600.html。配色 #2FDEAF/#383B4D/#5CA7FF、フォントRighteous、白い浮遊ピル型ナビ）を再現。`message.html`〜`requirement.html` は新卒採用マイクロサイト（配色 #009d7b系ティール、フォントKosugi Maru/M PLUS Rounded 1c、ヘッダーのロゴ+ドロップダウンナビ+2ボタン、波形の区切り・ドット柄装飾・オーガニックマスク写真・ピル型ボタン・グラデーション円アイコンなど）を再現。
- **「数字で知る」の実数値に差し替え**：従業員数199人・創業52年・平均年齢44歳（60歳以上再雇用者除く）・平均勤続年数18年・定着率85%（過去3年）・平均残業時間12時間/月・年間休日123日・有給休暇平均取得17日・女性管理職11%・育休取得率75%・育休復帰率100%など、実際に表示されていた数値をそのまま反映（以前は仮数値でした）。
- **「募集要項」の「選考方法」と「選考フロー」を区別**：テーブル内の「選考方法」は書類選考→適性検査→面接の3ステップ、別セクションの「選考フロー」はマイナビエントリー→会社説明会→ES提出→適性検査→面接2回→最終面接→内々定の7ステップ（色グラデーションのステップボックス）で、実サイトと同じ2つの情報として区別。
- **社長メッセージ全文・先輩社員2名分の実文言**を反映（残り4名分は簡略版のまま。既知のギャップとして各ページに注記）。

社員メッセージは実サイトで6名分掲載されていますが、本モックアップで実文言を確認できたのは2名分のみです。

## データ・デザインの正確性について（v3で追加修正）

さらにもう一度、全ページを実サイトと1:1で見比べて以下を修正しました。

- **`current/newgraduate.html` を新設**：`newgraduate_recruitment/`（マイクロサイトのトップページ）自体のページが抜けていたため追加。フルビジュアルヒーロー・斜めグラデーションのイントロ帯・6セクションのプレビューカードを再現し、各ページのロゴリンクもこのトップページに戻るよう修正（以前は本体サイトの採用ハブに直接戻ってしまっていました）。
- **フッターの配色を修正**：実際は背景ティール色＋白文字＋▶マーカー付きリンクでしたが、灰色背景で作っていたため差し替え。
- **`message.html`の先輩メッセージの文字が読めなかった問題を修正**：白文字が薄い背景に乗って視認性が低かったため、白背景カード＋濃色文字に変更。
- **`works.html`に「活躍できる人材」タブの内容を追加**：「求める人物像：自分の活躍するフィールドを積極的に拡げていく人材であってほしい」という実際の文章が丸ごと抜けていたため追加。
- **`requirement.html`の採用Q&Aを実際の13問に総入れ替え**：以前は5問の簡易版でしたが、実際は13問（文系選考・研修・DX・福利厚生・勤務地・休日出勤・在宅勤務・商工中金以外の仕事・社内の雰囲気・定着率・使用言語・英語スキル・残業）あり、使用言語も実際は「COBOL/PL1/VBA/アセンブラ/C#/VBScript/Java/C」と、これまで記載していたより多いことが判明したため反映。

## データ・デザインの正確性について（v4：コーポレートサイトTOPを追加）

`current/index.html`（採用情報ハブ＝pg600.html）と `newgraduate.html`（新卒採用マイクロサイトTOP）は再現していましたが、その2つが本来リンクする先である**本体サイトのルートページ（https://www.shochu-comp.co.jp/ 自体）**のページが存在していませんでした。各ページの「コーポレートサイトTOP」ボタンが誤って採用ハブに戻ってしまっていたのはこのためです。

- **`current/corporate.html` を新設**：ルートページを実際にブラウザで確認し、写真ヒーロー＋「NEWS」「MISSION」「RECRUIT」の3大見出しセクション（各見出しの頭文字だけ緑色にする実際のスタイル）、実際のお知らせ本文・MISSION文・RECRUITリード文を反映して再現。
- **全ページの「コーポレートサイトTOP」ボタンと、pg600ハブの「TOP」ナビ・フッター下部の「商工中金情報システム」リンクを `corporate.html` に向くよう修正**（以前はすべて採用ハブ`index.html`に自己参照していました）。

## 既知の簡略化

実サイトでは「福利厚生／研修制度／働く環境」「募集要項／採用Q&A」がそれぞれ別URLに分かれていますが、本モックアップでは関連ページを1ページにまとめています（`current/environment.html`、両サイトの `requirement.html`）。ページ数を増やしたい場合は、同じパターンで分割してください。
